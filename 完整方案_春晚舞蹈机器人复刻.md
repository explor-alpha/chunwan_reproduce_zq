```bash
# conda中环境变量配置(编译cpp扩展)
conda activate phmr

mkdir -p $CONDA_PREFIX/etc/conda/activate.d
mkdir -p $CONDA_PREFIX/etc/conda/deactivate.d

PHMR_PATH="/home/qunz/projects/clone/every-embodied/07-机器人操作、运动控制/Locomotion/video2robot/third_party/PromptHMR"

# activate
cat <<EOF > $CONDA_PREFIX/etc/conda/activate.d/env_vars.sh
# 保存旧变量
export OLD_PYTHONPATH=\$PYTHONPATH
export OLD_LD_LIBRARY_PATH=\$LD_LIBRARY_PATH
export OLD_PATH=\$PATH

# 项目路径
export PHMR_PATH=/home/qunz/projects/clone/every-embodied/07-机器人操作、运动控制/Locomotion/video2robot/third_party/PromptHMR
# 系统级CUDA路径
export CUDA_HOME=/usr/local/cuda-12.8

# (编译,有时运行也需要):nvcc
export PATH=\$CUDA_HOME/bin:\$PATH
# (运行)优先使用 pytorch 自带 runtime;有些项目会runtime JIT compile CUDA kernel所以第二优先nvcc
export LD_LIBRARY_PATH=$CONDA_PREFIX/lib/python3.10/site-packages/torch/lib:$CUDA_HOME/lib64:$LD_LIBRARY_PATH

# (PYTHONPATH):添加import Python库的搜索路径
export PYTHONPATH=\$PYTHONPATH:\$PHMR_PATH

EOF

# deactivate
cat <<EOF > $CONDA_PREFIX/etc/conda/deactivate.d/env_vars.sh
export PYTHONPATH=\$OLD_PYTHONPATH
unset OLD_PYTHONPATH

export LD_LIBRARY_PATH=\$OLD_LD_LIBRARY_PATH
unset OLD_LD_LIBRARY_PATH

export PATH=\$OLD_PATH
unset OLD_PATH

unset PHMR_PATH
unset CUDA_HOME
EOF
```

### Part 2 报错：原理-问题诊断-解决方式-检验

#### 报错1_OSError

> **1. （cpp扩展编译过程）编译cpp扩展前未安装cuda Toolkit（nvcc 编译器），会出现以下报错**
```
OSError: CUDA_HOME environment variable is not set. Please set it to your CUDA install root.
```

- **原理：**
	- 参考KeyNotes2（不再赘述）

- **问题诊断：**
```bash
which nvcc # 判断：确实无CUDA Toolkit（nvcc 编译器）
nvidia-smi # 判断：驱动版本
python -c "import torch; print(torch.version.cuda)" # 检查虚拟环境的cuda版本
```

- **解决方式：**
1. 准备：
	1. cpp基本环境：build-essential全家桶
	2. 对应win系统：英伟达驱动不要太旧
2. 判断安装什么版本的系统级CUDA Toolkit（nvcc 编译器）：
	1. 对于WSL，CUDA Toolkit装在Linux里，且不要带上驱动安装
	2. 对于5060Ti架构，对应sm_120，需要12.8以上的cuda支持
	3. 因为conda中报错前已经安装好了pytorch且可以运行，故可以保留pytorch，只需要系统级cuda和项目的pytorch版本匹配就行（或者小版本兼容）——运行：`python -c "import torch; print(torch.version.cuda)"`   (——我的输出为12.8)
	- （故我选择cuda 12.8）
3. 安装系统级的cuda 12.8：
```bash
# 进入临时目录下载
cd /tmp

# 下载 CUDA 12.8.0 运行文件 (针对 Linux x86_64)
# 注意：WSL 也可以使用标准的 Linux runfile，关键在于安装时“不选驱动”
wget https://developer.download.nvidia.com/compute/cuda/12.8.0/local_installers/cuda_12.8.0_570.86.10_linux.run

# 安装 CUDA Toolkit
sudo sh cuda_12.8.0_570.86.10_linux.run

# 交互：EULA 协议——accept；Driver——[ ]；CUDA Toolkit 12.8——[X]
```
- 验证
```bash
/usr/local/cuda-12.8/bin/nvcc -V
```

4. 配置cuda环境变量（我选择在conda里配置）
```
# 系统级CUDA路径
export CUDA_HOME=/usr/local/cuda-12.8

# (编译,有时运行也需要):nvcc
export PATH=\$CUDA_HOME/bin:\$PATH
```


#### 报错2_CUDA error: no kernel

> **2. 在初步配置完环境后运行项目：发现setup.py中未编译sm_120的sass（我的GPU框架必须要sm_120对应的kernal）**

- 初步配置完环境后，手动运行项目：
```bash
conda activate phmr
PROJ_PATH="/home/qunz/projects/clone/every-embodied/07-机器人操作、运动控制/Locomotion/video2robot"
pkill -f "$PROJ_PATH/visualization/robot_viser.py"
cd $PROJ_PATH

python scripts/extract_pose.py --project data/video_001
```
- 报错：
```
CUDA error: no kernel image is available for execution on the device
```

- **原理：**
	- 参考KeyNotes2（不再赘述）

- **问题诊断：**
	- `no kernel image` 可能有以下原因：
	1. 系统级cuda不可运行，无法编译此cpp扩展？——`/usr/local/cuda-12.8/lib64`可以发现我的`cuda12.8`是正在运行的
	2. cuda可运行，但是没有sm_120的kernal(我的GPU只能用sm_120的kernal)——(此时，如果我降低了系统cuda版本重新配置这个问题大概率依然存在)
	- 推测主要原因：
```
编译droidcalib的setup.py脚本不支持50系显卡的sm_120：

GPU: sm_120 (Compute Capability 12.0)
当前setup.py中 so: 只有 sm_60–sm_90
→ GPU 找不到可运行的 kernel
→ CUDA error: no kernel image is available for execution on the device
```

- **解决方案：**

1. 修改setup.py
- 打开video2robot/third_party/PromptHMR/pipeline/droidcalib/setup.py
- 修改
```
'nvcc': [
    '-O3',
    '-gencode=arch=compute_120,code=sm_120',
    '-gencode=arch=compute_120,code=compute_120',
    '-gencode=arch=compute_90,code=compute_90',
]
```

2. 清理旧编译：
- 进入phmr
- cd video2robot/third_party/PromptHMR/pipeline/droidcalib
- 执行：
```bash
rm -rf build
rm -rf *.egg-info
```
- 删除旧.so
```bash
rm $CONDA_PREFIX/lib/python3.10/site-packages/droid_backends_intr*.so
```

3. 重新编译：
```bash
export CPATH="$CONDA_PREFIX/include/eigen3:${CPATH:-}"

python setup.py install
```

- **检验：**
```bash
cuobjdump --list-elf $CONDA_PREFIX/lib/python3.10/site-packages/droid_backends_intr*.so | grep sm_
```
理想结果：sm120
```bash
python - << 'PY'
import torch
import droid_backends_intr

print("GPU:", torch.cuda.get_device_capability())
print("functions:", dir(droid_backends_intr))
PY
```
理想结果：没有报错

再次手动运行项目：
```bash
conda activate phmr
PROJ_PATH="/home/qunz/projects/clone/every-embodied/07-机器人操作、运动控制/Locomotion/video2robot"
pkill -f "$PROJ_PATH/visualization/robot_viser.py"
cd $PROJ_PATH

python scripts/extract_pose.py --project data/video_001
```
理想结果：顺利

#### 概述：phmr环境

**PromptHMR（视觉感知/眼睛）** Prompt-based Human Mesh Recovery：
1. 功能：看视频。当你给它一段人跳舞的视频时，它负责从 2D 的画面里，精准地识别出人的三维姿态、骨骼角度和体型（即 SMPL-X 模型）。
2. 为什么需要它：机器人看不懂像素视频，它需要知道人的手肘弯了多少度、膝盖抬了多高。PromptHMR 就是负责把“视频”变成“3D 数据”的工具。
3. 这是一个第三方的学术论文代码库，所以需要克隆到 third_party 目录下。
